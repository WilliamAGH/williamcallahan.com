<BackgroundInfo title="Overview: Key Components & Their Roles">
This guide covers setting up a modern Django project, including:

- **Django**: A Python web framework for rapid development and clean design.
- **Modern Template**: Using pre-built templates for a responsive UI/UX.
- **SaaS Core Functionality**: Implementing user authentication, admin panel, and core product models.

## Part 1: Setting Up Your Initial Django Project

This part covers setting up a standard, default Django project using your preferred workflow (Terminal, VS Code, or PyCharm).

### 1.1: Getting Started: Prerequisites & Setup

<ShellParentTabs>
  <ShellTab label="Python Installation" isDefault={true}>
    <CollapseDropdown summary="Ensure Python 3 is Installed">
      <div>You'll need Python 3 (ideally a recent version like 3.8 or newer). Check if Python is installed:</div>
      ```bash
      python3 --version
      ```
      <div>If not installed, or you need a different version:</div>
      <ul>
        <li><strong>Official Website</strong>: Download from <ExternalLink href="https://www.python.org/downloads/">python.org</ExternalLink>.</li>
        <li>
          Alternative for macOS (and Linux) - <ExternalLink href="https://brew.sh/">Homebrew</ExternalLink>:

          Use the following command:

          ```bash
          brew install python3
          ```
        </li>
        <li><strong>Linux</strong>: Use your distribution's package manager (e.g., <code>sudo apt update && sudo apt install python3</code> for Debian/Ubuntu).</li>
        <li><strong>Windows</strong>: Install from the Microsoft Store or python.org. Consider using Windows Subsystem for Linux (WSL).</li>
      </ul>
    </CollapseDropdown>
  </ShellTab>

  <ShellTab label='uv (recommended)'>

        <code>pip</code>, <code>pip-tools</code>, and <code>venv</code>.
      </div>
      <div>
        <strong>Installation Methods:</strong>
      </div>

      <ul>
        <li>
          <strong>Official Install Scripts (Cross-Platform - Preferred):</strong>
          <div>On macOS, Linux, and Windows (WSL/Git Bash/PowerShell):</div>
          ```bash
          curl -LsSf https://astral.sh/uv/install.sh | sh
          ```
          <div>
            Alternatively, for Windows PowerShell (if the above doesn't work
            directly):
          </div>
          ```bash
          irm https://astral.sh/uv/install.ps1 | iex
          ```
        </li>
        <li>
          <strong>Homebrew (macOS/Linux):</strong>{' '}
          ```bash
          brew install uv
          ```
        </li>
        <li>
          <strong>Cargo (if you have Rust installed):</strong>{' '}
          ```bash
          cargo install uv
          ```

        </li>
        <li>
          <strong>pip (globally or in a pre-existing venv):</strong>{' '}

      </ul>
      <div>

        <ExternalLink href='https://docs.astral.sh/uv/getting-started/installation/'>
          official <code>uv</code> installation guide.

      </div>
    </CollapseDropdown>
  </ShellTab>

  <ShellTab label="pip & venv">
    <CollapseDropdown summary="Setup pip & venv">

      <div>Or, to upgrade an existing pip version:</div>
      ```bash
      python3 -m pip install --upgrade pip
      ```


      <ul>

          ```bash
          # 1. Create project directory and setup environment
          mkdir my_django_project && cd my_django_project
          uv venv
          # This command creates a virtual environment in the .venv directory.
          # For more details on how uv manages Python environments, see the official documentation: https://docs.astral.sh/uv/pip/environments/

          # 2. Install Django
          uv add django

          # 3. Verify installation
          ```

## Part 2: Adding Your First App & Core Concepts

With the default project running, let's create your first Django app and understand core concepts.

<CollapseDropdown summary='Create Your First App & Configure'>
  <div className="py-[6px]">Django projects are composed of one or more "apps." An app is a self-contained module that handles a specific piece of functionality.</div>

  <div className="py-[6px]"><strong>1. Create the App:</strong></div>
  <InstructionToggleTabs>
    <InstructionTab label='uv' isDefault={true}>
      ```bash
      uv run manage.py startapp my_first_app
      ```
    </InstructionTab>
    <InstructionTab label='pip'>
      ```bash
      python manage.py startapp my_first_app
      ```
    </InstructionTab>
  </InstructionToggleTabs>

  <div className="py-[6px]"><strong>2. Register Your App:</strong></div>
  <div>Add your new app to <code>INSTALLED_APPS</code> in your project's <code>settings.py</code> file:</div>
  ```python
  INSTALLED_APPS = [
      # ... other apps
      'my_first_app', # Or 'my_first_app.apps.MyFirstAppConfig' if using AppConfig
      # ... django apps ...
  ]
  ```

  <div className="py-[6px]"><strong>3. Define a Simple Model:</strong></div>
  <div>Edit <code>my_first_app/models.py</code>:</div>
  ```python
  from django.db import models

  class Greeting(models.Model):
      message = models.CharField(max_length=255)
      created_at = models.DateTimeField(auto_now_add=True)

      def __str__(self):
          return self.message
  ```

  <div className="py-[6px]"><strong>4. Create & Apply Migrations:</strong></div>
  <InstructionToggleTabs>
    <InstructionTab label='uv' isDefault={true}>
      ```bash
      # Generate migration files
      uv run manage.py makemigrations my_first_app

      # Apply migrations to database
      uv run manage.py migrate
      ```
    </InstructionTab>
    <InstructionTab label='pip'>
      ```bash
      # Generate migration files
      python manage.py makemigrations my_first_app

      # Apply migrations to database
      python manage.py migrate
      ```
    </InstructionTab>
  </InstructionToggleTabs>
</CollapseDropdown>

<CollapseDropdown summary="Create Views and URL Configuration">
  <div><strong>1. Create a View:</strong></div>
  <div>Edit <code>my_first_app/views.py</code>:</div>
  ```python
  from django.http import HttpResponse
  from .models import Greeting

  def hello_world(request):
      greeting = Greeting.objects.first()  # Get the first greeting, or None
      if not greeting:
          greeting_text = "No greetings yet!"
      else:
          greeting_text = greeting.message
      return HttpResponse(f"<h1>{greeting_text}</h1><p>From my_first_app!</p>")
  ```

  <div><strong>2. Create App URLs:</strong></div>
  <div>Create <code>my_first_app/urls.py</code>:</div>
  ```python
  from django.urls import path
  from . import views

  urlpatterns = [
      path('hello/', views.hello_world, name='hello_world'),
  ]
  ```

  <div><strong>3. Include in Project URLs:</strong></div>
  <div>Update your project's <code>urls.py</code> (e.g., <code>config/urls.py</code>):</div>
  ```python
  from django.contrib import admin
  from django.urls import path, include

  urlpatterns = [
      path('admin/', admin.site.urls),
      path('app/', include('my_first_app.urls')),  # This adds your app URLs
  ]
  ```

  <div className="py-[6px]">Now visit <code>http://127.0.0.1:8000/app/hello/</code> to see your view in action.</div>
</CollapseDropdown>

<CollapseDropdown summary='Key Django Concepts'>
  <ul>
    <li><div><strong>Apps for Features:</strong> Organize functionality into reusable, modular apps</div></li>
    <li>
      <div><strong>MVT (Model-View-Template):</strong> Django's architecture pattern</div>
      <ul>
        <li><div>Models - Define data structure and interact with database</div></li>
        <li><div>Views - Handle logic and request/response flow</div></li>
        <li><div>Templates - Present information to users (HTML)</div></li>
      </ul>
    </li>
    <li><div><strong>URL Routing:</strong> Map web addresses to view functions</div></li>
    <li><div><strong>Admin Interface:</strong> Built-in data management UI</div></li>
    <li><div><strong>ORM:</strong> Object-Relational Mapper for database operations using Python</div></li>
  </ul>
</CollapseDropdown>

## Part 3: Level Up with Modern Project Templates

<BackgroundInfo title='Benefits of Modern Templates'>
These templates reduce boilerplate for common functionalities (auth, UI components, deployment setup), allowing you to focus on your unique application logic.
</BackgroundInfo>

<CollapseDropdown summary='Popular Modern Template Options'>
  <ul>
    <li>
      <strong><ExternalLink href="https://github.com/cookiecutter/cookiecutter-django">Cookiecutter Django</ExternalLink></strong>:
      <ul>
        <li><strong>Pros:</strong> Highly configurable, production-ready (Docker, Celery), many best practices.</li>
        <li><strong>Cons:</strong> Can be complex for beginners.</li>
      </ul>
    </li>
    <li>
      <strong><ExternalLink href="https://www.saaspegasus.com/">SaaS Pegasus</ExternalLink></strong>:
      <ul>
        <li><strong>Pros:</strong> For SaaS apps; includes subscriptions, teams, pre-built UI. Good documentation.</li>
        <li><strong>Cons:</strong> Paid.</li>
      </ul>
    </li>
    <li>
      <strong><ExternalLink href="https://github.com/saadmk11/djangox">DjangoX</ExternalLink></strong>:
      <ul>
        <li><strong>Pros:</strong> Simpler starter; Tailwind CSS, user auth.</li>
        <li><strong>Cons:</strong> Less feature-rich.</li>
      </ul>
    </li>
    <li>
      <strong><ExternalLink href="https://www.creative-tim.com/templates/django">Creative Tim</ExternalLink> / <ExternalLink href="https://themeforest.net/category/site-templates/admin-templates?term=django">ThemeForest</ExternalLink></strong>:
      <ul>
        <li><strong>Pros:</strong> Wide variety of themes.</li>
        <li><strong>Cons:</strong> Quality and Django integration vary.</li>
      </ul>
    </li>
  </ul>
</CollapseDropdown>

<CollapseDropdown summary="Setting Up a Modern Template">
  <div>Most templates provide detailed setup instructions, but here's a general workflow:</div>

  <div><strong>1. Generate or Clone the Template:</strong></div>
  <ShellParentTabs>
    <ShellTab label="Using Cookiecutter" isDefault={true}>
      <InstructionToggleTabs>
        <InstructionTab label="uv" isDefault={true}>
          ```bash
          # Install cookiecutter if needed
          uv add cookiecutter

          # Generate project from template
          uv run cookiecutter gh:cookiecutter/cookiecutter-django
          ```
        </InstructionTab>
        <InstructionTab label="pip">
          ```bash
          # Install cookiecutter if needed
          pip install cookiecutter

          # Generate project from template
          cookiecutter gh:cookiecutter/cookiecutter-django
          ```
        </InstructionTab>
      </InstructionToggleTabs>

    <ShellTab label="Using git clone">
      ```bash
      # Clone directly from repository
      git clone <repository_url> my_modern_project
      cd my_modern_project
      ```
    </ShellTab>
  </ShellParentTabs>

  <div><strong>2. Install Dependencies:</strong></div>
  <ShellParentTabs>
    <ShellTab label="Template with requirements.txt" isDefault={true}>
      <InstructionToggleTabs>
        <InstructionTab label="uv" isDefault={true}>
          ```bash

          uv venv

          ```
        </InstructionTab>
        <InstructionTab label="pip">
          ```bash

          python3 -m venv .venv
          source .venv/bin/activate  # On Windows: .\.venv\Scripts\activate

          # Install dependencies
          pip install -r requirements.txt
          ```
        </InstructionTab>
      </InstructionToggleTabs>
    </ShellTab>
    <ShellTab label="Template with pyproject.toml">
      <InstructionToggleTabs>

        <InstructionTab label="PDM">

    </ShellTab>

  <div>Adjust environment variables or settings according to template documentation.</div>

  <div><strong>4. Run Initial Setup/Migrations:</strong></div>
  <InstructionToggleTabs>
    <InstructionTab label="uv" isDefault={true}>
      ```bash
      uv run manage.py migrate
      ```
    </InstructionTab>
    <InstructionTab label="pip">

      ```bash
      python manage.py migrate
      ```
    </InstructionTab>
    <InstructionTab label="Docker">

      ```bash
      docker-compose exec web python manage.py migrate
      ```

    </InstructionTab>
  </InstructionToggleTabs>

## Part 4: Implementing a SaaS Core with SQL Database

<BackgroundInfo title='Core SaaS Features in Django'>
Django offers robust features for SaaS applications:
- **User Authentication**: `django.contrib.auth` for user management.
- **Admin Panel**: `django.contrib.admin` for data management.
- **ORM & Models**: Django's ORM for database interaction via Python.
</BackgroundInfo>

<CollapseDropdown summary='Setting Up SQL Database'>
  <div><strong>1. Choose a Database:</strong></div>
  <div>PostgreSQL or MySQL are recommended for production. SQLite (default) works for development.</div>

  <div><strong>2. Install Database Drivers:</strong></div>
  <ShellParentTabs>
    <ShellTab label="PostgreSQL" isDefault={true}>
      <InstructionToggleTabs>
        <InstructionTab label="uv" isDefault={true}>
          ```bash
          uv add psycopg2-binary
          ```
        </InstructionTab>
        <InstructionTab label="pip">
          ```bash
          pip install psycopg2-binary
          ```
        </InstructionTab>
      </InstructionToggleTabs>
    </ShellTab>
    <ShellTab label="MySQL">
      <InstructionToggleTabs>
        <InstructionTab label="uv" isDefault={true}>
          ```bash
          uv add mysqlclient
          ```
        </InstructionTab>
        <InstructionTab label="pip">
          ```bash
          pip install mysqlclient
          ```
        </InstructionTab>
      </InstructionToggleTabs>
    </ShellTab>
  </ShellParentTabs>

  <div><strong>3. Configure Database in settings.py:</strong></div>
  <div>Update the <code>DATABASES</code> dictionary in <code>settings.py</code>:</div>

  <ShellParentTabs>
    <ShellTab label="PostgreSQL" isDefault={true}>
      ```python
      DATABASES = {
          'default': {
              'ENGINE': 'django.db.backends.postgresql',
              'NAME': 'mydatabase',
              'USER': 'mydatabaseuser',
              'PASSWORD': 'mypassword',
              'HOST': '127.0.0.1',
              'PORT': '5432',
          }
      }
      ```
    </ShellTab>
    <ShellTab label="MySQL">
      ```python
      DATABASES = {
          'default': {
              'ENGINE': 'django.db.backends.mysql',
              'NAME': 'mydatabase',
              'USER': 'mydatabaseuser',
              'PASSWORD': 'mypassword',
              'HOST': '127.0.0.1',
              'PORT': '3306',
          }
      }
      ```
    </ShellTab>
  </ShellParentTabs>

  <BackgroundInfo title='Database Credentials Tip'>
    For production, use environment variables for database credentials, not hardcoded values:
    ```python
    DATABASES = {
        'default': {
            'ENGINE': 'django.db.backends.postgresql',
            'NAME': os.environ.get('DB_NAME', 'mydatabase'),
            'USER': os.environ.get('DB_USER', 'mydatabaseuser'),
            'PASSWORD': os.environ.get('DB_PASSWORD', 'mypassword'),
            'HOST': os.environ.get('DB_HOST', '127.0.0.1'),
            'PORT': os.environ.get('DB_PORT', '5432'),
        }
    }
    ```
  </BackgroundInfo>
</CollapseDropdown>

<CollapseDropdown summary='Admin Panel Configuration'>
  <div>Django's admin panel provides a powerful interface for managing your application's data.</div>

  <div><strong>1. Register Models in admin.py:</strong></div>
  <div>In your app's <code>admin.py</code> file:</div>
  ```python
  from django.contrib import admin
  from .models import Greeting

  @admin.register(Greeting)
  class GreetingAdmin(admin.ModelAdmin):
      list_display = ('message', 'created_at')
      search_fields = ('message',)
  ```

  <div><strong>2. Access Admin Interface:</strong></div>
  <div>After starting the development server, visit <code>http://127.0.0.1:8000/admin/</code> and log in with superuser credentials.</div>

  <div><strong>Admin Customization Options:</strong></div>
  <ul>
    <li><code>list_display</code>: Fields to show in list view</li>
    <li><code>search_fields</code>: Fields to search through</li>
    <li><code>list_filter</code>: Enable filtering by fields</li>
    <li><code>fieldsets</code>: Organize fields in detail view</li>
    <li><code>readonly_fields</code>: Fields that can't be edited</li>
  </ul>
</CollapseDropdown>

## Part 6: Troubleshooting Common Issues

<CollapseDropdown summary="Development Server Issues">
  <div><strong>Stopping the Server:</strong></div>
  <ul>
    <li><div><strong>Clean Stop:</strong> <code>Ctrl+C</code> in the terminal</div></li>
    <li><div><strong>If <code>Ctrl+Z</code> was used:</strong> Use <code>fg</code> then <code>Ctrl+C</code>, or <code>jobs</code> and <code>kill %JOB_NUMBER</code></div></li>
  </ul>

  <div><strong>Address Already in Use:</strong></div>
  <div>If you see "Address already in use," find and kill the process:</div>
  ```bash
  # Find the process using port 8000
  lsof -i :8000

  # Kill the process
  kill <PID>
  ```
</CollapseDropdown>

<CollapseDropdown summary='Migration Issues'>
  <div><strong>"No changes detected":</strong></div>
  <ul>
    <li>Check that your app is in <code>INSTALLED_APPS</code></li>
    <li>Verify you saved all changes to <code>models.py</code></li>
    <li>Ensure you're specifying the correct app name in the command</li>
  </ul>

  <div><strong>Migration Errors:</strong></div>
  <ul>
    <li>Verify database connection settings</li>
    <li>Check for syntax errors in models</li>
    <li>View migration status with <code>python manage.py showmigrations</code></li>
  </ul>
</CollapseDropdown>

<CollapseDropdown summary='Static Files & Templates'>
  <div><strong>Static Files Not Loading (404s):</strong></div>
  <ul>
    <li><strong>Development:</strong> Check <code>STATIC_URL</code>, ensure files are in <code>app_name/static/app_name/</code></li>
    <li><strong>Production:</strong> Run <code>collectstatic</code>, configure web server for <code>STATIC_ROOT</code></li>
  </ul>

  <div><strong><code>TemplateDoesNotExist</code> Error:</strong></div>
  <ul>
    <li>Verify app is in <code>INSTALLED_APPS</code></li>
    <li>Check template path - should be in <code>app_name/templates/app_name/</code></li>
    <li>Check for typos in template names</li>
  </ul>
</CollapseDropdown>

<CollapseDropdown summary='Import & Package Issues'>
  <div><strong>ModuleNotFoundError:</strong></div>
  <ul>
    <li>Ensure virtual environment is activated</li>
    <li>Check if package is installed:</li>
  </ul>

  <InstructionToggleTabs>
    <InstructionTab label="uv" isDefault={true}>
      ```bash
      # Install package
      uv add <package_name>

      # Verify installed packages
      uv pip list
      ```
    </InstructionTab>
    <InstructionTab label="pip">
      ```bash
      # Install package
      pip install <package_name>

      # Verify installed packages
      pip list
      ```
    </InstructionTab>
  </InstructionToggleTabs>

  <div><strong>Other Import Issues:</strong></div>
  <ul>
    <li>Check for typos in import statements</li>
    <li>Verify proper Python package structure</li>
    <li>Beware of circular imports</li>
  </ul>
</CollapseDropdown>

<CollapseDropdown summary='Database Debugging'>
  <div><strong>Connection Problems:</strong></div>
  <ul>
    <li>Verify database settings in <code>settings.py</code></li>
    <li>Ensure database server is running</li>
    <li>Check firewall settings</li>
    <li>Confirm correct driver is installed</li>
  </ul>

  <div><strong>Using Django Shell for Debugging:</strong></div>
  <InstructionToggleTabs>
    <InstructionTab label="uv" isDefault={true}>
      ```bash
      uv run manage.py shell
      ```
    </InstructionTab>
    <InstructionTab label="pip">

    </InstructionTab>
  </InstructionToggleTabs>

</CollapseDropdown>
