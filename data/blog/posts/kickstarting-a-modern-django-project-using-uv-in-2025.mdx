This guide covers setting up a modern Django project, including:

- **Modern Template**: Using pre-built templates for a responsive UI/UX.

## Part 1: Project Foundation & Environment Setup

To begin, ensure you have Python 3 installed. You'll then need a package manager to handle project dependencies. We recommend `uv` for its speed and modern features, but traditional `pip` with `venv` is also covered.

<CollapseDropdown summary="0.1: Installing `uv` (Recommended Package Manager)">
  - `uv` is a fast Python package installer and resolver, written in Rust.
  - **Installation**:
    - **macOS and Linux (Shell)**:
      ```bash
      curl -LsSf https://astral.sh/uv/install.sh | sh
      ```
    - **Windows (PowerShell)**:
      ```powershell
      irm https://astral.sh/uv/install.ps1 | iex
      ```
    - **Other Methods**: `uv` can also be installed via `pipx`, `pip`, Homebrew, Cargo, WinGet, Scoop, or by downloading binaries directly from [GitHub Releases](https://docs.astral.sh/uv/getting-started/installation/#github-releases). See the [official `uv` installation guide](https://docs.astral.sh/uv/install/) for all options.
  - After installation, you might need to restart your terminal or add `uv`'s installation directory to your PATH. Verify with `uv --version`.
</CollapseDropdown>

<BackgroundInfo title="Option 1: Using `uv` (Recommended)">
The following sections detail setting up your project using `uv`.
</BackgroundInfo>

<CollapseDropdown summary="1.1: Python Version Management with `uv` (Optional)">
  - While a system Python 3 is sufficient, `uv` can also install and manage specific Python versions for your projects.
  - **Install Latest Python with `uv`**:
    ```bash
    uv python install
    ```
  - **Install a Specific Python Version**:
    ```bash
    uv python install 3.11 # Or your desired version
    ```
  - **List `uv`-managed Python installations**:
    ```bash
    uv python list
    ```
  - `uv` will use a managed Python or an existing system Python if compatible. See the [uv Python documentation](https://docs.astral.sh/uv/guides/install-python/).
</CollapseDropdown>

<CollapseDropdown summary="1.2: Virtual Environments with `uv`">
  - **Purpose**: Isolate project dependencies.
  - **Creating and Activating a Virtual Environment**:
    ```bash
    # Navigate to your project directory (e.g., my_django_project)
    # mkdir my_django_project && cd my_django_project
    uv venv
    ```
  - This creates a `.venv` directory and activates it if your shell is supported. Otherwise, activate manually (e.g., `source .venv/bin/activate`).
</CollapseDropdown>

<CollapseDropdown summary="1.3: Installing Django with `uv` (Project-based)">
  - For a `uv`-managed project (typically using a `pyproject.toml` file), use `uv add` to include Django as a dependency. If you haven't initialized your project, you can do so with `uv init`.
  - With an active `uv` virtual environment in your project directory:
    ```bash
    uv add django
    ```
  - This command updates your `pyproject.toml` file, adding Django to your project dependencies. See [Managing dependencies with `uv`](https://docs.astral.sh/uv/concepts/projects/dependencies/#adding-dependencies) for details.
  - To install a specific version or set constraints:
    ```bash
    uv add "django>=5.0,<5.1" # Example version constraint
    ```
  - Verify Django is available: `django-admin --version` (after `uv sync` if needed, or by running commands via `uv run django-admin --version`).
  <BackgroundInfo title="Quick Install (Non-Project based)">
  If you just want to quickly install Django into an active virtual environment without managing it through `pyproject.toml` yet, you can use:
  `uv pip install django`
  This is similar to using traditional `pip`.
  </BackgroundInfo>
</CollapseDropdown>

<BackgroundInfo title="Option 2: Using `pip` & `venv` (Alternative)">
If you prefer to use `pip` and `venv` directly, follow these steps.
</BackgroundInfo>

<CollapseDropdown summary="1.4: Checking Python & `pip` (for `pip` users)">
  - **Verify Python 3 Installation**:
    ```bash
    python3 --version
    # Or on some systems
    python --version
    ```
  - **Ensure `pip` is Available** (usually included with Python 3.4+):
    ```bash
    pip3 --version
    # Or
    pip --version
    ```
  - If Python 3 is not installed, download it from [python.org](https://www.python.org/).
</CollapseDropdown>

<CollapseDropdown summary="1.5: Virtual Environments with `venv` (for `pip` users)">
  - **Purpose**: Isolate project dependencies.
  - **Creating a Virtual Environment**:
    ```bash
    # Navigate to your project directory
    # mkdir my_django_project && cd my_django_project
    python3 -m venv .venv # Or your preferred name like 'env'
    ```
  - **Activating the Virtual Environment**:
    - **macOS/Linux**:
      ```bash
      source .venv/bin/activate
      ```
    - **Windows**:
      ```bash
      .\.venv\Scripts\activate
      ```
</CollapseDropdown>

<CollapseDropdown summary="1.6: Installing Django with `pip` (for `pip` users)">
  - With your `venv` virtual environment activated:
    ```bash
    pip install django
    ```
  - To install a specific version:
    ```bash
    pip install django==5.0 # Or desired version
    ```
  - Verify: `django-admin --version`
</CollapseDropdown>

## Part 2: Choosing a Modern Template

Using a template accelerates development and provides a professional UI.

<BackgroundInfo title="Benefits of Templates">
Templates offer pre-built UI/UX, responsive design, and common functionalities like authentication. I think this is important so you don't spend all your time on the boilerplate (you avoid "reinventing the wheel") and can spend your time focusing on what you can innovate on!
</BackgroundInfo>

<CollapseDropdown summary="2.1: Popular Template Options">
  - **[Cookiecutter Django](https://github.com/cookiecutter/cookiecutter-django)**:
    - **Pros**: Highly configurable, production-ready (Docker, Caddy/Traefik, Celery, etc.), includes many best practices. Great for building robust applications.
    - **Cons**: Can be overwhelming for beginners due to its many options.
  - **[SaaS Pegasus](https://www.saaspegasus.com/)**:
    - **Pros**: Specifically designed for SaaS applications, includes features like subscriptions, teams, and pre-built UI components. Offers extensive documentation and support.
    - **Cons**: Paid product.
  - **[Creative Tim](https://www.creative-tim.com/templates/django)** / **[ThemeForest](https://themeforest.net/category/site-templates/admin-templates?term=django)**:
    - **Pros**: Wide variety of admin and frontend themes, often visually appealing.
    - **Cons**: Quality and Django integration can vary. May require more effort to integrate with Django's backend logic.
  - **[DjangoX](https://github.com/saadmk11/djangox)**:
    - **Pros**: A simpler starter template with a focus on modern frontend (Tailwind CSS) and essential features like user auth.
    - **Cons**: Less feature-rich than Cookiecutter or Pegasus but easier to get started with.
</CollapseDropdown>

<CollapseDropdown summary="2.2: Integrating Your Chosen Template">
  - **General Steps**:
    1.  **Download/Clone**: Obtain the template files, often via Git or by running a project generator command (like with Cookiecutter).
    2.  **Follow Documentation**: Most templates come with specific setup instructions. Pay close attention to these.
    3.  **Install Dependencies**:
        - If the template provides a `requirements.txt`:
          ```bash
          # To install into your active uv environment:
          uv pip install -r requirements.txt
          ```
        - <BackgroundInfo title="Importing to `pyproject.toml`">
          If you are integrating this template into a `uv`-managed project (with `pyproject.toml`), you can import these dependencies by running:
          `uv add -r requirements.txt`
          This will add the dependencies from `requirements.txt` to your `pyproject.toml` file. See the [`uv add` documentation](https://docs.astral.sh/uv/concepts/projects/dependencies/#importing-dependencies).
          </BackgroundInfo>
        - If the template already uses `pyproject.toml` and a lock file (e.g., `uv.lock`), you might use `uv sync` to install dependencies.
    4.  **Configure Settings**: Adjust `settings.py` for your database, static files, etc., as per the template's guidelines.
    5.  **Run Initial Migrations**:
        ```bash
        python manage.py migrate
        ```

</CollapseDropdown>

## Part 3: Implementing a SaaS Core with SQL Database

A SaaS application requires user management, an admin interface, and a database.

<BackgroundInfo title="Core SaaS Features in Django">
Django provides these with minimal configuration:
- **User Authentication**: `django.contrib.auth` for user accounts, groups, permissions, sessions.
- **Admin Panel**: `django.contrib.admin` auto-generates an admin interface.
- **ORM & Models**: Django's ORM simplifies defining data structure and database interaction.
</BackgroundInfo>

<CollapseDropdown summary="3.1: Django's User Model and Admin">
  - **`django.contrib.auth`**: Handles user registration, login, logout, password hashing, permissions.
  - **Customizing User Model**: Extend `AbstractUser` or `AbstractBaseUser` for needs like email as username or profile fields. Refer to Django's [customizing authentication documentation](https://docs.djangoproject.com/en/stable/topics/auth/customizing/).
  - **Django Admin**: Available at `/admin/`. Register models to manage their data.
</CollapseDropdown>

<CollapseDropdown summary="3.2: Database Setup (PostgreSQL or MySQL)">
  - **PostgreSQL/MySQL**: Offer robustness and scalability for production over SQLite.
  - **Install Database Drivers (using `uv`)**:
    - **PostgreSQL**:
      ```bash
      uv pip install psycopg2-binary
      ```
    - **MySQL**:
      ```bash
      uv pip install mysqlclient
      ```
  - **Configure `settings.py`**: Update the `DATABASES` setting.
  <BackgroundInfo title="Database Credentials">
  Never hardcode sensitive credentials directly in `settings.py` for production. Use environment variables (e.g., with `python-decouple` or `django-environ`).
  </BackgroundInfo>
</CollapseDropdown>

<CollapseDropdown summary="3.3: Defining Core Data Models (`models.py`)">
  - Define your application's data structure in `your_app_name/models.py`.
  - **Example**: A simple SaaS product might have models for `Tenant`, `Subscription`, `ProductFeature`.
    ```python
    # In your_app_name/models.py
    from django.db import models
    from django.contrib.auth.models import User # Or your custom user model

    class SubscriptionPlan(models.Model):
        name = models.CharField(max_length=100)
        price = models.DecimalField(max_digits=6, decimal_places=2)
        # ... other fields

        def __str__(self):
            return self.name

    class Tenant(models.Model):
        name = models.CharField(max_length=200)
        owner = models.ForeignKey(User, on_delete=models.CASCADE)
        subscription_plan = models.ForeignKey(SubscriptionPlan, null=True, blank=True, on_delete=models.SET_NULL)
        created_at = models.DateTimeField(auto_now_add=True)

        def __str__(self):
            return self.name
    ```
  - **Running Migrations**: After defining or changing models, create and apply database migrations:
    ```bash
    python manage.py makemigrations your_app_name
    python manage.py migrate
    ```
</CollapseDropdown>

<CollapseDropdown summary="3.4: Admin Panel Integration (`admin.py`)">
  - Register your models in `your_app_name/admin.py` to make them manageable via the Django admin interface.
    ```python
    # In your_app_name/admin.py
    from django.contrib import admin
    from .models import Tenant, SubscriptionPlan

    @admin.register(Tenant)
    class TenantAdmin(admin.ModelAdmin):
        list_display = ('name', 'owner', 'subscription_plan', 'created_at')
        list_filter = ('subscription_plan', 'created_at')
        search_fields = ('name', 'owner__username')

    @admin.register(SubscriptionPlan)
    class SubscriptionPlanAdmin(admin.ModelAdmin):
        list_display = ('name', 'price')

    # Alternatively, without decorator:
    # admin.site.register(Tenant, TenantAdmin)
    # admin.site.register(SubscriptionPlan, SubscriptionPlanAdmin)
    ```
</CollapseDropdown>

## Part 4: IDE Setup: VSCode & IntelliJ IDEA (PyCharm)

A well-configured IDE boosts Django development productivity.

<BackgroundInfo title="IDE Productivity Benefits">
Features like code completion, debugging, Django tools, database integration, and version control accelerate development.
</BackgroundInfo>

<CollapseDropdown summary="4.1: Setup for Microsoft VS Code">
  - **Essential Extensions**:
    - **Python (Microsoft)**: Core Python support.
    - **Django (Batiste Plantation)**: Django snippets, commands, utilities.
    - **Pylance (Microsoft)**: Language server for Python.
  - **Configuration (`.vscode/settings.json`)**:
    ```json
    {
        "python.defaultInterpreterPath": ".venv/bin/python", // Path to uv created venv python
        "python.linting.flake8Enabled": true,
        "python.formatting.provider": "black", // Assuming Black formatter
        "editor.formatOnSave": true,
        "python.linting.flake8Args": ["--max-line-length=88"],
        "[python]": {
            "editor.defaultFormatter": "ms-python.black-formatter"
        }
    }
    ```
  - **Debugging (`.vscode/launch.json`)**: Configuration for the Django development server.
</CollapseDropdown>

<CollapseDropdown summary="4.2: Setup for IntelliJ IDEA (PyCharm)">
  - **PyCharm Professional**: Has comprehensive, out-of-the-box Django support.
  - **IntelliJ IDEA Ultimate with Python Plugin**: Can also be configured for robust Django development.
  - **Creating/Importing a Django Project**:
    - PyCharm will often auto-detect a Django project or allow you to create one with Django support enabled.
    - Ensure the correct Python interpreter (from your virtual environment) is selected for the project.
  - **Run/Debug Configurations**:
    - PyCharm typically creates a "Django server" run configuration automatically. You can customize it in **Run > Edit Configurations**.
  - **Key Features**:
    - Smart code completion for Django templates, models, views, and URLs.
    - Integrated database tools.
    - Visual debugger.
    - Manage.py task execution from the IDE.
</CollapseDropdown>

## Part 5: Kickstarting Development & Fast-Track Resources

With the project and IDE set up, begin development.

<CollapseDropdown summary="5.1: Essential `manage.py` Commands">
  - `python manage.py runserver`: Starts the development server.
  - `python manage.py createsuperuser`: Creates an admin user.
  - `python manage.py startapp <app_name>`: Creates a new Django app within your project.
  - `python manage.py makemigrations [app_name]`: Creates new migration files based on model changes.
  - `python manage.py migrate [app_name] [migration_name]`: Applies migrations to the database.
  - `python manage.py shell`: Opens an interactive Python shell with your Django project loaded.
  - `python manage.py check`: Checks for common problems in your project.
  - `python manage.py collectstatic`: Collects static files into one directory for deployment.
</CollapseDropdown>

<CollapseDropdown summary="5.2: Structuring Your Django Project">
  - **Apps for Features**: Organize your project into reusable "apps." Each app should handle a distinct piece of functionality (e.g., `users`, `products`, `blog`).
  - **Templates**: Store HTML templates in a `templates` directory, typically within each app or at the project level.
  - **Static Files**: CSS, JavaScript, and images go into a `static` directory, also within each app or at the project level.
  - **Media Files**: User-uploaded files are typically handled by `MEDIA_ROOT` and `MEDIA_URL` settings.
</CollapseDropdown>

<CollapseDropdown summary="5.3: Key Django Concepts to Master">
  - **MVT (Model-View-Template) Architecture**: Understand how data (Model), logic (View), and presentation (Template) interact.
  - **URL Routing (`urls.py`)**: How Django maps web requests to your view functions or classes.
  - **Views (Function-Based Views & Class-Based Views)**: The logic that handles requests and returns responses.
  - **Django ORM & QuerySets**: Interacting with your database using Python objects.
  - **Forms (`forms.py`)**: Handling user input, validation, and rendering HTML forms.
  - **Django Templates Language (DTL)**: Syntax for embedding dynamic content in HTML.
</CollapseDropdown>

<CollapseDropdown summary="5.4: Deploying Your Application (_Brief Overview_)">
  - **Considerations**: `DEBUG = False`, static file serving, WSGI server (Gunicorn), web server (Nginx), database configuration, environment variables.
  - **Popular Platforms**:
    - PaaS: Heroku, PythonAnywhere, Google App Engine, AWS Elastic Beanstalk.
    - IaaS/VPS: DigitalOcean, Linode, AWS EC2.
    - Containerization: Docker with Docker Compose or Kubernetes.
  <BackgroundInfo title="Deployment is a Topic of Its Own">
  Deployment involves many steps beyond Django. Refer to specific platform documentation and Django deployment guides. Many starter templates (e.g., Cookiecutter Django) include deployment configurations.
  </BackgroundInfo>
</CollapseDropdown>
