## Part 1: Setting Up Your Initial Django Project

This part covers setting up a standard, default Django project. Choose one of the following methods based on your preferred workflow (Terminal, VS Code, or PyCharm). All methods achieve the same goal: a basic, running Django project.

### 1.1: Prerequisites

- **Python 3**: Ensure you have a recent version of Python 3 installed. Check with `python3 --version` or `python --version`.
- **Package Manager**: You'll need a way to install Python packages. We strongly recommend `uv` for its speed and features. `pip` (which comes with Python) is the traditional alternative.

<CollapseDropdown summary="Installing `uv` (Recommended)">
  - If you don't have `uv`:
    ```bash
    # macOS/Linux
    curl -LsSf https://astral.sh/uv/install.sh | sh
    # Windows PowerShell
    irm https://astral.sh/uv/install.ps1 | iex
    ```
  - Verify: `uv --version`. See [official `uv` installation guide](https://docs.astral.sh/uv/install/) for more methods.
</CollapseDropdown>

### 1.2: Method 1: Terminal Setup (`uv` / `pip` + `django-admin`)

This method uses only terminal commands for setup.

<CollapseDropdown summary="Step 1: Create Project Directory & Environment (uv Recommended)">
  ```bash
  # Navigate to parent directory
  mkdir my_django_project && cd my_django_project
  # Create/activate venv using uv
  uv venv
  # Initialize uv project (creates pyproject.toml)
  uv init
  ```
  (If venv doesn't activate, use `source .venv/bin/activate` or `.\.venv\Scripts\activate`).
</CollapseDropdown>

<CollapseDropdown summary="Step 2: Install Django (uv Recommended)">
  ```bash
  # Add Django to pyproject.toml and install
  uv add django
  # Verify
  uv run django-admin --version
  ```
</CollapseDropdown>

<CollapseDropdown summary="(Alternative) Step 1 & 2 using `pip` & `venv`">
  ```bash
  # Navigate to parent directory
  mkdir my_django_project && cd my_django_project
  # Create venv using standard Python
  python3 -m venv .venv
  # Activate venv (macOS/Linux: source .venv/bin/activate | Windows: .\.venv\Scripts\activate)
  # Install Django using pip
  pip install django
  # Verify
  django-admin --version
  ```
</CollapseDropdown>

<CollapseDropdown summary="Step 3: Create Default Django Project Files">
  - Regardless of how you installed Django (uv or pip), use `django-admin` to create the project files. Use `.` to create in the current directory.
  ```bash
  # Using uv run (if uv project exists)
  uv run django-admin startproject config .

  # Or directly (if venv active)
  django-admin startproject config .
  ```
  (Replace `config` with your preferred project configuration directory name).
  - This creates `manage.py` and the `config/` directory (`settings.py`, `urls.py`, etc.).
</CollapseDropdown>

<CollapseDropdown summary="Step 4: Run Initial Migrations & Dev Server">
  - Apply the initial database migrations (creates necessary tables for auth, etc.):
  ```bash
  # Using uv run
  uv run manage.py migrate
  # Or directly
  python manage.py migrate
  ```
  - Run the development server:
  ```bash
  # Using uv run
  uv run manage.py runserver
  # Or directly
  python manage.py runserver
  ```
  - Access at `http://127.0.0.1:8000/`. Stop server with `Ctrl+C`.
</CollapseDropdown>

<CollapseDropdown summary="Step 5: Create Superuser (Optional but Recommended)">
  - To use the admin panel later:
  ```bash
  # Using uv run
  uv run manage.py createsuperuser
  # Or directly
  python manage.py createsuperuser
  ```
  - Follow prompts for username/password.
</CollapseDropdown>

This method uses VS Code's features to assist with managing the default project created via terminal or opened from existing code.

## Part 2: Adding Your First App & Core Concepts

With the default project running, let's create your first Django app to understand core concepts like apps, models, views, and URLs.

<CollapseDropdown summary="5.1: Create Your First App (Terminal)">
  - Django projects are composed of one or more "apps." An app is a self-contained module that handles a specific piece of functionality (e.g., a blog, polls, user accounts).
  - From your project root (where `manage.py` is), with your virtual environment active:
    ```bash
    # Using uv
    uv run manage.py startapp my_first_app

    # Or directly
    python manage.py startapp my_first_app
    ```
    (Replace `my_first_app` with a suitable name, following Python identifier rules: lowercase, underscores, no hyphens).
  - This creates a new directory `my_first_app/` with files like `models.py`, `views.py`, `apps.py`, etc.
</CollapseDropdown>

<CollapseDropdown summary="5.2: Register Your App & Basic Model">
  - **Register App**: Add your new app to `INSTALLED_APPS` in your project's `settings.py` file (e.g., `your_project_name/settings.py`):
    ```python
    INSTALLED_APPS = [
        # ... other apps
        'my_first_app', # Or 'my_first_app.apps.MyFirstAppConfig' if using AppConfig
        # ... django apps ...
    ]
    ```
  - **Define a Simple Model**: Edit `my_first_app/models.py`:
    ```python
    from django.db import models

    class Greeting(models.Model):
        message = models.CharField(max_length=255)
        created_at = models.DateTimeField(auto_now_add=True)

        def __str__(self):
            return self.message
    ```
</CollapseDropdown>

<CollapseDropdown summary="5.3: Create Migrations for Your App (Terminal)">
  - After defining or changing models in your app:
    ```bash
    # Using uv
    uv run manage.py makemigrations my_first_app

    # Or directly
    python manage.py makemigrations my_first_app
    ```
  - Then apply the migrations:
    ```bash
    # Using uv
    uv run manage.py migrate

    # Or directly
    python manage.py migrate
    ```
</CollapseDropdown>

<CollapseDropdown summary="5.4: Basic View and URL Configuration">
  - **Create a View**: Edit `my_first_app/views.py`:
    ```python
    from django.http import HttpResponse
    from .models import Greeting

    def hello_world(request):
        greeting = Greeting.objects.first() # Get the first greeting, or create one
        if not greeting:
            greeting_text = "No greetings yet!"
        else:
            greeting_text = greeting.message
        return HttpResponse(f"<h1>{greeting_text}</h1><p>From my_first_app!</p>")
    ```
  - **App URLs**: Create `my_first_app/urls.py`:
    ```python
    from django.urls import path
    from . import views

    urlpatterns = [
        path('hello/', views.hello_world, name='hello_world'),
    ]
    ```
  - **Project URLs**: Include your app's URLs in your project's `urls.py` (e.g., `your_project_name/urls.py`):
    ```python
    from django.contrib import admin
    from django.urls import path, include

    urlpatterns = [
        path('admin/', admin.site.urls),
        path('app/', include('my_first_app.urls')), # Example path for your app
    ]
    ```
  - Now, if your server is running, you should be able to visit `http://127.0.0.1:8000/app/hello/`.
</CollapseDropdown>

<CollapseDropdown summary="5.5: Project Structuring & Key Concepts Recap">
  - **Apps for Features**: Key to Django is organizing functionality into reusable apps.
  - **MVT (Model-View-Template)**: Django follows this pattern. Models interact with data, Views handle logic and request/response, Templates present information.
  - **URL Routing**: `urls.py` files map web addresses to views.
  - **Templates**: HTML files (we used a direct `HttpResponse` here for simplicity, but typically you'd use Django templates).
  - **Static Files & Media**: For CSS, JS, images, and user uploads (covered more with advanced templates or specific setup).
</CollapseDropdown>

## Part 3: Level Up with Modern Project Templates

While the default Django project is great for learning, starting new projects often benefits from feature-rich templates that provide modern UI, CI/CD setup, SaaS features, and more out-of-the-box.

<BackgroundInfo title="Benefits of Modern Templates">
These templates reduce boilerplate for common functionalities (auth, UI components, deployment setup), allowing you to focus on your unique application logic.
</BackgroundInfo>

<CollapseDropdown summary="6.1: Popular Modern Template Options">
  - **[Cookiecutter Django](https://github.com/cookiecutter/cookiecutter-django)**:
    - **Pros**: Highly configurable, production-ready (Docker, Celery), many best practices.
    - **Cons**: Can be complex for beginners.
  - **[SaaS Pegasus](https://www.saaspegasus.com/)**:
    - **Pros**: For SaaS apps; includes subscriptions, teams, pre-built UI. Good documentation.
    - **Cons**: Paid.
  - **[Creative Tim](https://www.creative-tim.com/templates/django)** / **[ThemeForest](https://themeforest.net/category/site-templates/admin-templates?term=django)**:
    - **Pros**: Wide variety of themes.
    - **Cons**: Quality and Django integration vary.
  - **[DjangoX](https://github.com/saadmk11/djangox)**:
    - **Pros**: Simpler starter; Tailwind CSS, user auth.
    - **Cons**: Less feature-rich.
</CollapseDropdown>

<CollapseDropdown summary="6.2: Integrating a Modern Template (Terminal Focus)">
  - **General Terminal Steps for Integration**:
    1.  **Generate/Clone Template** (Terminal):
        ```bash
        # Example using Cookiecutter Django
        # uv run cookiecutter gh:cookiecutter/cookiecutter-django

        # Example using git clone (for other templates)
        # git clone <repository_url> my_modern_project
        # cd my_modern_project
        ```
        Follow the specific template's instructions for generation or cloning.
    2.  **Follow Documentation**: Template-specific setup is crucial.
    3.  **Install Dependencies (Terminal)**:
        - Check the template's documentation for dependency management (it might use `uv`, `pip`, `poetry`, `pdm`, or Docker).
        - **If `requirements.txt`**:
          ```bash
          # Using uv (recommended, if creating a new .venv)
          # uv venv
          # uv pip install -r requirements.txt
          # Or if importing into existing uv project:
          # uv add -r requirements.txt

          # Using pip (if creating new .venv)
          # python3 -m venv .venv && source .venv/bin/activate
          # pip install -r requirements.txt
          ```
        - **If `pyproject.toml` (uv/poetry/pdm)**:
          ```bash
          # uv sync
          # poetry install
          # pdm install
          ```
        - **If Docker**: Follow the template's `docker-compose up --build` or similar instructions.
    4.  **Configure Settings**: Adjust `.env` files or `settings.py` as per template docs.
    5.  **Run Initial Setup/Migrations (Terminal)**:
        - Templates often have specific initial setup commands.
        - Common command:
          ```bash
          # Using uv (if applicable)
          # uv run manage.py migrate
          # Or directly
          # python manage.py migrate
          # Or via Docker
          # docker-compose exec web python manage.py migrate
          ```
        Refer to the template's documentation for exact commands.
</CollapseDropdown>

## Part 4: Implementing a SaaS Core with SQL Database

This involves terminal commands for setup and migrations, alongside Python code for models and admin configuration. The Django admin panel itself is a web interface.

<BackgroundInfo title="Core SaaS Features in Django">
Django offers robust features for SaaS applications:
- **User Authentication**: `django.contrib.auth` for user management.
- **Admin Panel**: `django.contrib.admin` for data management (web interface).
- **ORM & Models**: Django's ORM for database interaction via Python.
</BackgroundInfo>

<CollapseDropdown summary="5.1: Django User Model & Admin Interface">
  - **User Model (`django.contrib.auth`)**: Handles user accounts, login, permissions. For custom needs (e.g., email as username), extend `AbstractUser`/`AbstractBaseUser`. See [Django auth docs](https://docs.djangoproject.com/en/stable/topics/auth/customizing/).
  - **Admin Interface**: Web-based panel at `/admin/`. Register models to manage their data here (see 3.4).
</CollapseDropdown>

<CollapseDropdown summary="5.2: Database Setup (Terminal & Config)">
  - **Database Choice**: PostgreSQL or MySQL recommended for production.
  - **Install Drivers (Terminal)**:
    - Activate your virtual environment first.
    - **PostgreSQL**:
      ```bash
      # Using uv (recommended)
      uv add psycopg2-binary
      # Or using pip
      # pip install psycopg2-binary
      ```
    - **MySQL**:
      ```bash
      # Using uv (recommended)
      uv add mysqlclient
      # Or using pip
      # pip install mysqlclient
      ```
  - **Configure `settings.py`**: Update `DATABASES` with your credentials (refer to Django docs).
  <BackgroundInfo title="Database Credentials Tip">
  For production, use environment variables for database credentials, not hardcoded values.
  </BackgroundInfo>
</CollapseDropdown>

<CollapseDropdown summary="5.3: Models & Migrations (Code & Terminal)">
  - **Define Models**: Code your models in `your_app_name/models.py`.
  - **Create & Apply Migrations (Terminal)**:
    - After defining or changing models, run these commands from your project root (with virtual environment active):
    - **Create Migrations**:
      ```bash
      # Using uv (recommended)
      uv run manage.py makemigrations your_app_name
      # Or directly
      # python manage.py makemigrations your_app_name
      ```
    - **Apply Migrations**:
      ```bash
      # Using uv (recommended)
      uv run manage.py migrate
      # Or directly
      # python manage.py migrate
      ```
</CollapseDropdown>

<CollapseDropdown summary="5.4: Admin Panel Setup (Code & Web)">
  - **Register Models**: In `your_app_name/admin.py`, register models for the admin web interface.
  - **Accessing Admin**: After creating a superuser (Part 5), access via browser at `/admin/`.
<CollapseDropdown summary="5.4: Admin Panel Integration (Code & Web Interface)">
  - **Register Models**: In `your_app_name/admin.py`, register your models to make them manageable via the Django admin web interface (example code provided in previous versions).
  - **Accessing Admin**: Once models are registered and you have created a superuser (see Part 5), access the admin panel through your browser at the `/admin/` URL of your running development server.
</CollapseDropdown>

## Part 6: Diagnosing Common Issues

Troubleshooting tips for common Django and environment problems.

<CollapseDropdown summary="8.1: Stopping Dev Server">
  - **Clean Stop**: `Ctrl+C` in the server terminal for graceful shutdown.
  - **`Ctrl+Z` (Unix-like systems)**: Suspends, not stops. Use `fg` then `Ctrl+C`, or `jobs` and `kill %<job_id>`.
</CollapseDropdown>

<CollapseDropdown summary="8.2: Migration Issues">
  - **`makemigrations` shows "No changes detected"**: Check `INSTALLED_APPS`, saved `models.py`, or app name if specified.
  - **`migrate` errors**: Check database connection; resolve inconsistent history (cautiously).
  - **View migration status**: `uv run manage.py showmigrations`.
</CollapseDropdown>

<CollapseDropdown summary="8.3: Static Files Not Loading (404s)">
  - **Development (`DEBUG = True`)**: Check `STATIC_URL`, ensure files in `app_name/static/app_name/` or dirs in `STATICFILES_DIRS`.
  - **Production (`DEBUG = False`)**: Run `collectstatic`; configure web server for `STATIC_ROOT`.
</CollapseDropdown>

<CollapseDropdown summary="8.4: `TemplateDoesNotExist`">
  - Check template path in error. Ensure app in `INSTALLED_APPS`. Templates in `app_name/templates/app_name/` or `TEMPLATES[0]['DIRS']`. Check for typos.
</CollapseDropdown>

<CollapseDropdown summary="8.5: Import Errors (`ModuleNotFound` etc.)">
  - **Check venv**: Ensure virtual environment is active.
  - **Package installed?**: `uv add <pkg>` (for `pyproject.toml`) or `uv pip install <pkg>`. Verify with `uv pip list`.
  - **Typos/Circular Imports**: Review import statements and module dependencies.
</CollapseDropdown>

<CollapseDropdown summary="8.6: Database Connection Problems">
  - Verify `DATABASES` in `settings.py`. Ensure DB server is running/accessible. Check firewall. Confirm driver installed.
</CollapseDropdown>

<CollapseDropdown summary="8.7: Django Shell for Debugging">
  - Test queries, models, utilities: `uv run manage.py shell`.
  - Import models (e.g., `from my_app.models import MyModel`) and test.
</CollapseDropdown>

<CollapseDropdown summary="8.8: Checking `uv` Environment Info">
  - **Active venv details**: `uv venv` (from project root).
  - **List installed packages**: `uv pip list`.
</CollapseDropdown>
