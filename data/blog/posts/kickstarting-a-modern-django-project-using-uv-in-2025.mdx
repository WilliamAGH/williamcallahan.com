## Part 2: Adding Your First App & Core Concepts

A Django **project** is the overall container for your application's settings and configurations (e.g., database settings, installed apps, URL routing at the project level). A project can contain multiple **apps**, which are modular components that handle specific functionalities (like a blog, a user management system, etc.). Think of the project as the building and apps as the distinct rooms or services within it.

With the default project running, let's create your first Django app and understand core concepts.

<CollapseDropdown summary='Create your first app & configure'>
  <div className="py-[6px]">Django projects are composed of one or more "apps." An app is a self-contained module that handles a specific piece of functionality.</div>

  <div className="py-[6px]"><strong>1. Create the App:</strong></div>
  <div className="instruction-tabs">
    <InstructionToggleTabs>
      <InstructionTab label="uv">

```bash
uv run manage.py startapp my_first_app
```

      </InstructionTab>
      <InstructionTab label="pip">

```bash
python manage.py startapp my_first_app
```

      </InstructionTab>
    </InstructionToggleTabs>
  </div>

  <BackgroundInfo title='App naming tip'>
    App labels need to be valid Python identifiers. Stick to lowercase letters and optional underscores (for example, `my_first_app`) so the generated package imports cleanly.
  </BackgroundInfo>

  <div className="py-[6px]"><strong>2. Register Your App:</strong></div>
  <div>Add your new app to <code>INSTALLED_APPS</code> in your project's <code>settings.py</code> file:</div>
  ```python
  INSTALLED_APPS = [
      # ... other apps
      'my_first_app', # Or 'my_first_app.apps.MyFirstAppConfig' if using AppConfig
      # ... django apps ...
  ]
  ```

  <div className="py-[6px]"><strong>3. Define a Simple Model:</strong></div>
  <div>Edit <code>my_first_app/models.py</code>:</div>
  ```python
  from django.db import models

  class Greeting(models.Model):
      message = models.CharField(max_length=255)
      created_at = models.DateTimeField(auto_now_add=True)

      def __str__(self):
          return self.message
  ```

  <div className="py-[6px]"><strong>4. Create & Apply Migrations:</strong></div>
  <div className="instruction-tabs">
    <InstructionToggleTabs>
      <InstructionTab label="uv">

```bash
# Generate migration files
uv run manage.py makemigrations my_first_app

# Apply migrations to database
uv run manage.py migrate
```

      </InstructionTab>
      <InstructionTab label="pip">

```bash
# Generate migration files
python manage.py makemigrations my_first_app

# Apply migrations to database
python manage.py migrate
```

      </InstructionTab>
    </InstructionToggleTabs>
  </div>
</CollapseDropdown>

<CollapseDropdown summary="Create views and URL configuration">
  <div><strong>1. Create a View:</strong></div>
  <div>Edit <code>my_first_app/views.py</code>:</div>
  ```python
  from django.http import HttpResponse
  from .models import Greeting

  def hello_world(request):
      greeting = Greeting.objects.first()  # Get the first greeting, or None
      if not greeting:
          greeting_text = "No greetings yet!"
      else:
          greeting_text = greeting.message
      return HttpResponse(f"<h1>{greeting_text}</h1><p>From my_first_app!</p>")
  ```

  <div><strong>2. Create App URLs:</strong></div>
  <div>Create <code>my_first_app/urls.py</code>:</div>
  ```python
  from django.urls import path
  from . import views

  urlpatterns = [
      path('hello/', views.hello_world, name='hello_world'),
  ]
  ```

  <div><strong>3. Include in Project URLs:</strong></div>
  <div>Update your project's <code>urls.py</code> (e.g., <code>config/urls.py</code>):</div>
  ```python
  from django.contrib import admin
  from django.urls import path, include

  urlpatterns = [
      path('admin/', admin.site.urls),
      path('app/', include('my_first_app.urls')),  # This adds your app URLs
  ]
  ```

  <div className="py-[6px]">Now visit <code>http://127.0.0.1:8000/app/hello/</code> to see your view in action.</div>
</CollapseDropdown>

<CollapseDropdown summary='Key Django concepts'>
  <ul>
    <li><div><strong>Apps for Features:</strong> Organize functionality into reusable, modular apps</div></li>
    <li>
      <div><strong>MVT (Model-View-Template):</strong> Django's architecture pattern</div>
      <ul>
        <li><div>Models - Define data structure and interact with database</div></li>
        <li><div>Views - Handle logic and request/response flow</div></li>
        <li><div>Templates - Present information to users (HTML)</div></li>
      </ul>
    </li>
    <li><div><strong>URL Routing:</strong> Map web addresses to view functions</div></li>
    <li><div><strong>Admin Interface:</strong> Built-in data management UI</div></li>
    <li><div><strong>ORM:</strong> Object-Relational Mapper for database operations using Python</div></li>
  </ul>
</CollapseDropdown>

## Part 4: Implementing Core Admin Features with SQL Database

Django has built-in features for web applications:
- **User Authentication**: `django.contrib.auth` for user management.
- **Admin Panel**: `django.contrib.admin` for data management.

  <div className="macos-frame-tabs">

  <BackgroundInfo title='Database Credentials Tip'>
    For production, use environment variables for database credentials, not hardcoded values:
    ```python

            'PASSWORD': os.environ.get('DB_PASSWORD', 'mypassword'),
            'HOST': os.environ.get('DB_HOST', '127.0.0.1'),
            'PORT': os.environ.get('DB_PORT', '5432'),
        }
    }
    ```

  from django.contrib import admin

  <div>After starting the development server, visit <code>http://127.0.0.1:8000/admin/</code> and log in with superuser credentials.</div>

  <div><strong>Admin Customization Options:</strong></div>
  <ul>
    <li><code>list_display</code>: Fields to show in list view</li>
    <li><code>search_fields</code>: Fields to search through</li>
    <li><code>list_filter</code>: Enable filtering by fields</li>
    <li><code>fieldsets</code>: Organize fields in detail view</li>
    <li><code>readonly_fields</code>: Fields that can't be edited</li>
  </ul>
</CollapseDropdown>

## Beyond Development: Deployment Considerations

While this guide focuses on setting up your local development environment, deploying your Django application to a production server involves additional steps. Unlike the built-in development server, production deployments typically use a Web Server Gateway Interface (WSGI) server (like Gunicorn or uWSGI) to handle requests, often in conjunction with a web proxy server (like Nginx or Apache, or AWS Lambda).

For more on deployment strategies and options, I made a recent post sharing my current hosting preferences: <ExternalLink href="/blog/how-i-host-my-apps-how-to-deploy-in-2025">How I Host My Apps: How to Deploy in 2025</ExternalLink>

## Part 6: Troubleshooting Common Issues

<CollapseDropdown summary="Development server issues">
  <div><strong>Stopping the Server:</strong></div>
  <ul>
    <li><div><strong>Clean Stop:</strong> <code>Ctrl+C</code> in the terminal</div></li>
    <li><div><strong>If <code>Ctrl+Z</code> was used:</strong> Use <code>fg</code> then <code>Ctrl+C</code>, or <code>jobs</code> and <code>kill %JOB_NUMBER</code></div></li>
  </ul>

  <div><strong>Address Already in Use:</strong></div>
  <div>If you see "Address already in use," find and kill the process:</div>
  ```bash
  # Find the process using port 8000
  lsof -i :8000

  # Kill the process
  kill <PID>
  ```
</CollapseDropdown>

<CollapseDropdown summary='Migration issues'>
  <div><strong>"No changes detected":</strong></div>
  <ul>
    <li>Check that your app is in <code>INSTALLED_APPS</code></li>
    <li>Verify you saved all changes to <code>models.py</code></li>
    <li>Ensure you're specifying the correct app name in the command</li>
  </ul>

  <div><strong>Migration Errors:</strong></div>
  <ul>
    <li>Verify database connection settings</li>
    <li>Check for syntax errors in models</li>
    <li>View migration status with <code>python manage.py showmigrations</code></li>
  </ul>
</CollapseDropdown>

<CollapseDropdown summary='Static files & templates'>
  <div><strong>Static Files Not Loading (404s):</strong></div>
  <ul>
    <li><strong>Development:</strong> Check <code>STATIC_URL</code>, ensure files are in <code>app_name/static/app_name/</code></li>
    <li><strong>Production:</strong> Run <code>collectstatic</code>, configure web server for <code>STATIC_ROOT</code></li>
  </ul>

  <div><strong><code>TemplateDoesNotExist</code> Error:</strong></div>
  <ul>
    <li>Verify app is in <code>INSTALLED_APPS</code></li>
    <li>Check template path - should be in <code>app_name/templates/app_name/</code></li>
    <li>Check for typos in template names</li>
  </ul>
</CollapseDropdown>
